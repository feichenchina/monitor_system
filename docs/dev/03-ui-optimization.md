# 前端 UI/UX 优化与性能提升日志

## 1. 需求分析
- 优化机器列表显示，增强卡状态可读性。
- 解决后台自动刷新导致的 DOM 闪烁。
- 提升整体视觉效果，增强输入框等交互组件。

## 2. 技术方案与核心逻辑
- **Smart Merge (智能差量更新) 机制**: 
    - **背景**: 传统的定时全量替换 `machines.value = newData` 会导致 Vue 重新渲染所有表格行，引起视觉上的“闪烁”，并丢失用户的输入焦点或滚动位置。
    - **核心逻辑**: 在 `fetchMachines` 中遍历新数据，对于已存在的机器记录，仅对比并更新其状态字段（如 `status`, `gpu_info`），而保持对象引用不变。这利用了 Vue 的响应式系统，仅触发受影响 DOM 节点的局部更新，极大地提升了用户体验。
- **UI/UX 设计哲学**:
    - **极简主义**: 合并冗余列（如将“状态”标签融入“IP 地址”文字颜色），减少视觉噪音。
    - **反馈连贯性**: 移除后台刷新时的全屏 Loading，仅在首次加载或手动刷新时显示。
- **性能优化手段**:
    - **虚拟 DOM 辅助**: 为 `<el-table>` 绑定 `row-key="id"`。在执行 Smart Merge 时，稳定的 `row-key` 能帮助 Vue 的 Diff 算法精确识别复用节点，减少重绘开销。
    - **Favicon 的极致兼容性**: 将 SVG 转换为 Base64 内联，不仅消除了对文件服务的网络请求依赖，还规避了部分浏览器在非 HTTPS 环境下对 `.svg` 文件的 MIME 类型解析限制。

## 3. 实现细节与代码见解
- **表格布局微调**:
    - 通过 `.el-table .cell { overflow: visible !important; }` 解决 Element Plus 默认 `hidden` 导致 `el-badge` 上标被裁切的问题。
    - 针对加速卡状态列使用 `min-width` 而非固定 `width`，以适应不同分辨率下的卡信息堆叠。
- **视觉色彩规范**:
    - 在线 (Online): 使用 `#67C23A` (Success Green)，加粗显示以突出核心可用性。
    - 离线 (Offline): 使用 `#909399` (Secondary Grey)，低对比度处理以减少对用户注意力的干扰。
    - 错误 (Error): 使用 `#F56C6C` (Danger Red)，通过颜色对比传达警示信息。
