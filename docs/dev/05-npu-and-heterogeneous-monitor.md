# 昇腾 NPU 优化与异构机器监控日志

## 1. 需求分析
- 优化昇腾 NPU 状态解析，解决无任务显存占用高导致误判的问题。
- 支持 NVIDIA GPU + Huawei NPU 异构环境监控。
- 提升 SSH 状态检测性能。

## 2. 技术方案与架构设计
- **异构状态感知算法 (Heterogeneous Awareness)**:
    - **背景**: 昇腾 NPU 在初始化后即便无任务也会占用约 500MB-1GB 显存，传统的“显存占比 > 5%”阈值判断会导致大量误报。
    - **方案**: 引入进程级检查。通过解析 `npu-smi info` 输出中的进程列表（Process Table）。若列表中存在非系统占用的 PID，则判定为 `Busy`；若显示 "No running processes found"，即便有显存占用也判定为 `Idle`。
- **SSH 管道吞吐量优化**:
    - **瓶颈**: 每次 SSH 连接/握手耗时约 200ms-500ms。原先检测一台机器需要 4 次独立命令，总耗时接近 2s。
    - **方案**: 命令合并执行。将 `uname -m`, `cat /etc/os-release`, `nvidia-smi`, `npu-smi` 通过 `&&` 和自定义分隔符 `|||SECTION|||` 拼接。后端只需一次 SSH 调用即可获取全量原始数据，将单机检测延迟降低了约 70%。

## 3. 实现细节与容错机制
- **正则解析鲁棒性**:
    - 针对 NVIDIA 和 Huawei 不同的输出格式，编写了容错率更高的正则表达式。
    - 解决了 `nvidia-smi` 在某些驱动版本下输出空行导致的计数偏差问题（计数逻辑从行数统计改为解析对象统计）。
- **异构结果聚合**:
    - 设计了统一的加速卡详情模型，将不同厂家的显存、温度、健康度数据标准化，方便前端统一渲染。
- **实时输出 (Raw Output) 重构**:
    - `get_raw_monitor` 接口不仅返回合并后的命令输出，还增加了“检测节点”标记，帮助运维人员快速定位 SSH 通信或权限的具体失败环节。

## 4. 验证
- 编写测试脚本 `test_npu_parsing.py` 验证 Idle/Busy 解析正确。
- 异构混合插卡机器能同时显示两类卡状态。
- 检测延迟显著降低。
