# PCIe 拓扑可视化集成日志

## 1. 需求分析
- 集成 `PCIETopoPainter` 工具，实现机器 PCIe 拓扑图的自动化生成与可视化。
- 支持在前端详情页沉浸式展示拓扑图。
- 实现启动自动刷新与数据持久化。

## 2. 技术方案
- **远程采集**: `topo_service.py` 负责打包工具、远程执行并解析结果。
- **数据持久化**: `Machine` 模型新增 `pci_topo_json` (Text) 字段，存储 JSON 数据。
- **前端渲染**: 基于 `Cytoscape.js` + `cytoscape-dagre` 渲染拓扑图。

## 3. 实现细节
- **后端拓扑服务**:
    - 实现 `update_all_machines_topo` 全量刷新。
    - 使用 `python3 -m zipfile` 远程解压。
    - 增加 `--all` 参数，防止 `lspci` 输出兼容性导致节点过度过滤。
- **前端组件**:
    - `PCIeTopo.vue`: 实现节点样式优化（水平文字、白色描边）。
    - 独立弹窗展示: `App.vue` 新增 `topoDialogVisible`。
    - **沉浸式 UI (2026-03-18 更新)**: 
        - 移除详情页中的 Tab 入口，改为列表页按钮直达。
        - 弹窗改为**全屏模式** (`fullscreen`)。
        - 背景设为深色半透明 (`rgba(0, 0, 0, 0.85)`)，增强沉浸感。
        - 右上角保留显式关闭按钮 (X)，并美化为圆形悬浮样式。
    - **UI 优化与美化 (2026-03-18 第二轮)**:
        - 解决关闭按钮与图例重叠问题：将图例和控制按钮移至左上角 (`top: 24px; left: 24px`)，关闭按钮保留在右上角。
        - 调整背景透明度：从 `0.85` 降至 `0.15` 并添加 `backdrop-filter: blur(8px)`，提升视觉层次感。
        - 美化图例：改为深色半透明玻璃风格 (`rgba(30, 30, 30, 0.85)`)，适配暗色背景，增加阴影和圆角细节。
        - 美化关闭按钮：增加悬停旋转缩放动画，提升交互体验。
    - **渲染与交互增强 (2026-03-18 第三轮)**:
        - **线条增强**: 将所有连接线的宽度加倍 (`width: (original || 1) * 2`)，确保在深色背景下清晰可见。
        - **文字增强**: 节点和线条文字颜色适配深色模式（浅色文字+深色描边），字号分别增大至 `14px` 和 `12px`，并加粗。
        - **缩放优化**: 设置 `wheelSensitivity: 0.5`（默认值通常较小），显著提升鼠标滚轮缩放的响应速度和幅度。
    - **子模块适配与新特性支持 (2026-03-19)**:
        - **核心工具升级**: 更新 `libs/PCIETopoPainter` 子模块的 commit 节点，以支持新的拓扑解析能力。
        - **图例扩展**: 在 `PCIeTopo.vue` 的图例中新增了 `Storage`（存储设备）的颜色映射 (`#FFA07A`)，并在画布上相应渲染。
        - **多重连线支持**: 修复了两个节点之间存在多条连接（如多个 PCIe 链路）时 Cytoscape 渲染冲突的问题。通过引入 `edgeIdCounter` 为每条边生成唯一标识符（如 `${source}-${target}_2`）解决。
        - **UI 细节调整**: 将节点文字的最大宽度 (`text-max-width`) 从 200px 增加至 260px，以容纳更长的设备描述。
        - **后端与配置变更**: 调整 `backend/main.py` 默认端口从 `8000` 到 `9000`；更新 `frontend/package.json` 版本号至 `1.1.1`。
    - **自动刷新机制优化 (2026-03-19 第二轮)**:
        - **联动刷新**: 修改了后端的 `/machines/{id}/refresh` 和 `/machines/refresh_all` 接口，使得用户在点击【刷新】和【刷新全部机器】按钮时，**总是同步触发**对应的拓扑图更新任务 (`update_machine_topo`)。
        - **前端响应式预览**: 在 `PCIeTopo.vue` 中引入了组件生命周期内的后台轮询机制（每 5 秒自动请求一次 `/topo` 接口）。通过字符串比对 `JSON.stringify` 判断拓扑数据是否发生实质性变更，仅在变更时重新渲染 Cytoscape 实例，实现了用户预览拓扑图时的**无感同步刷新**。

## 4. 踩坑记录
- **数据库迁移**: SQLite 不支持 `ALTER TABLE ... ADD COLUMN` 添加非空约束，需谨慎处理默认值。
- **远程执行**: 初始使用 `unzip`，但部分环境缺失。已改为 `python3 -m zipfile`。
- **lspci 兼容性**: 在部分机器上 `prune_tree` 逻辑误杀所有节点。通过强制开启 `--all` 参数解决。
- **Cytoscape**: `width: label` 已弃用。连线文字默认跟随路径旋转，阅读困难。通过 `text-rotation: none` 强制水平。
- **构建错误**: `App.vue` 添加 CSS 时引入语法错误（未正确闭合）。已修复。
- **事件冲突**: 点击“拓扑”按钮时，若焦点在输入框（如备注），`blur` 事件优先触发导致保存提示，而按钮 `click` 事件未能触发弹窗。
    - **修复**: 在按钮上添加 `@mousedown.prevent`，阻止点击时的默认失焦行为，确保弹窗逻辑执行。
- **运行时错误**: `ReferenceError: topoDialogVisible is not defined`。
    - **原因**: `script setup` 中遗漏了 `topoDialogVisible` 的定义，导致模板渲染和方法调用时无法访问该变量。
    - **修复**: 在 `script setup` 中补充 `const topoDialogVisible = ref(false);`。
